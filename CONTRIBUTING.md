# Contributing

Read the README, About, and Roadmap before opening a change.

Use an Issue before making a large rule, format, scoring, or integration change. Keep each pull request focused and include the problem, intended result, examples, validation steps, compatibility impact, and documentation updates.

Rule changes should include examples that pass, examples that fail, clear diagnostic text, known limitations, and automated fixtures.

Use clear commit prefixes such as `feat:`, `fix:`, `docs:`, `test:`, `refactor:`, and `chore:`.

Contributors are responsible for correctness, licensing, and review of submitted material. Do not publish private information or material that you do not have permission to distribute.
