# Security Policy

This repository is in the foundation phase and has not published a production-ready release.

Do not include private files or sensitive technical details in public Issues. Use GitHub private reporting when available. Otherwise open a minimal Issue requesting a private communication channel.

Reports should identify the affected commit, input type, reproduction steps, observed impact, and a sanitized example.

Sensitive areas include file-path handling, configuration loading, report output, optional integrations, and unexpected disclosure of analyzed content.

Validated reports will be reviewed privately where practical. Public notes should identify affected versions and corrections without exposing user information.
